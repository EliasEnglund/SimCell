# Cleanup Policy

Every milestone must include cleanup. The project should optimize for long-term development, not temporary speed.

Required cleanup at the end of each milestone:

- Delete obsolete prototype code, scenes, and assets.
- Rename unclear concepts before more code depends on them.
- Remove duplicated simulation rules from UI and presentation scripts.
- Update docs when gameplay assumptions change.
- Keep reference assets separate from runtime assets.
- Prefer small, understandable systems over speculative abstractions.

Dead code should not be archived in the project. If old behavior matters, capture it in docs or version control history.
