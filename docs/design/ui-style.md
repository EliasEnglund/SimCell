# UI Style

The UI should support repeated play, not look like a landing page.

## Layout Principles

- Primary workspace in the center.
- Side panels only when they add direct action or inspection.
- Compact top resource bar.
- Left-side navigation dock with image-only view icons.
- Hover popups for detail instead of permanent clutter.

## Panels

Panels should be dark, slightly translucent, and edged with controlled cyan glow. Avoid flat dull gray.

Lists should be dense and readable. The membrane outside molecule list should feel closer to a Finder-style list than large block cards.

The main navigation should share the same dark/cyan language as the top resource bar. Avoid gray footer bars behind the view icons.

## Interaction Feedback

- Hover: subtle glow or row highlight.
- Selection: stronger outline or filled highlight.
- Dragging: lift, soft shadow, and snap feedback.
- Disabled: dim but still readable.
- Build buttons: visible only when the action is relevant.
